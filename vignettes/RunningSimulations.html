Running Simulations for Evolution of Transposons

Anindya Sharma

2021-02-02

The retrocombinator package provides a method for simulating the molecular evolutionary process of retrotransposon recombination over large scales of time.

To use it, load the library first.

library(retrocombinator)

Simulations can be done in two ways - with and without flags for inactivity. The functions can be called in the following way:

# Calling a simulation without flags
simulate_without_flags_random(num_seq = 5, seq_length = 20,
                       point_mutation_model = "K80",
                       num_jumps = 20, timestep = 2,
                       burst_probability = 0.1, burst_mean = 1,
                       recomb_mean = 1,
                       file_out = "without_flags_vignette.out",
                       num_out_init = 10, num_out_tags = 10,
                       num_out_seqs = 10, num_out_pair = 10,
                       to_seed = TRUE, seed = 0)

Similarly, we can run a simulation with flags.

# Calling a simulation without flags
simulate_with_flags_random(num_seq = 10, 
                           seq_length = 200,
                    point_mutation_model = "K80",
                    num_sensitive_posns = 10, 
                    inactive_probability = 0.1, 
                    num_jumps = 50, 
                    timestep = 2,
                    burst_probability = 0.1, 
                    burst_mean = 1,
                    recomb_mean = 1,
                    selection_threshold = 60, 
                    fam_proportion = 0.7, 
                    fam_percentage = 70,
                    file_out = "with_flags_vignette.out",
                    num_out_init = 10, 
                    num_out_tags = 10,
                    num_out_seqs = 10, 
                    num_out_pair = 10,
                    to_seed = TRUE, 
                    seed = 0)

Again the results of the simulation can be analysed using the input_data function.
